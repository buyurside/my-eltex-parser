# Описание
Скрипт создан с целью сборки конфигурации с маршрутизаторов [Eltex ESR-3100](https://eltex-co.ru/catalog/service_gateways/servisnyy_marshrutizator_esr-3100/), поскольку до этого были безуспешные попытки подружить их с [oxidized](https://github.com/ytti/oxidized).

## Принцип работы
1. Посылаем по ssh команду `show running-config`
2. Парсим вывод, отрезаем голову до начала конфига (`hostname ...`)
3. Проверяем, есть ли файлы в целевой директории, куда собираемся сохранить спаршенный конфиг
4. Если есть файлы в директории, то сверяем последний с нашим выводом
5. Если полученный с железки вывод и последний сохраненный конфиг одинаковы, то пропускаем этот хост
6. Если есть изменения (вывод и последний конфиг не одинаковы), сохраняем конфиг

## Конфиг
Скриптик питается `yaml`-конфигами.

Ниже приведу примеры конфигов для разных условий.

_Конфиг для авторизации по ключу:_
```
hosts:
  - { "ip": 10.123.12.34, "port": 22, "hostname": "ROUTER-1" }
  - { "ip": 10.123.12.35, "port": 22, "hostname": "ROUTER-2" }
username: "user"
```

_Для авторизации по кредам:_
```
hosts:
  - { "ip": 10.123.12.34, "port": 22, "hostname": "ROUTER-1" }
username: "user"
password: "password"
```


### Также поддерживаются следующие переменные
Можно указывать в списке параметров хоста директорию, куда будут складываться конфиги:
```
# переменная dir
hosts:
  - { "ip": 10.123.12.34, "port": 22, "hostname": "ROUTER-1", dir: "My-ELTEX-dir" }
```
Рабочая директория:
```
working_dir: "/srv/eltex_backups"
```
Пауза в секундах между запросами к оборудованию:
```
delay: 123
```
